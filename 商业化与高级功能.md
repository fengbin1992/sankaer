# 第六阶段：商业化与高级功能

> 本文档为"三卡二"项目第六阶段开发任务清单。
> 目标：增加支付、社交、防作弊、数据统计，准备正式上线。
> 依赖：第五阶段（平台适配）已完成。

---

## 任务清单

### 一、登录系统完善

- [ ] **1.1** 实现微信授权登录
  - 微信小游戏 wx.login → 后端 code2Session → 创建/绑定用户
  - 📎 参考：[三卡二小程序.md - §9.1 登录方式](../三卡二小程序.md#91-登录方式)

- [ ] **1.2** 实现手机号 + 验证码登录
  - 短信验证码发送（阿里云 SMS）
  - 验证码校验 + 用户创建/登录
  - 📎 参考：[三卡二小程序.md - §9.1 + §9.2 统一账号体系](../三卡二小程序.md#92-统一账号体系)

- [ ] **1.3** 实现跨端账号绑定
  - 手机号绑定微信（合并数据）
  - 同一账号互踢机制（同时只能一端在线）
  - 📎 参考：[三卡二小程序.md - §9.2 统一账号体系](../三卡二小程序.md#92-统一账号体系)

### 二、支付系统

- [ ] **2.1** 实现微信小游戏支付
  - 充值档位（6/30/98/298/648 元）
  - 支付下单 → 微信支付 → 回调确认 → 到账
  - 📎 参考：[三卡二小程序.md - §10.1 支付方式矩阵 + §10.2 充值档位](../三卡二小程序.md#101-支付方式矩阵)

- [ ] **2.2** 实现支付回调接口
  - `server/api/handler/payment_callback.go`
  - 签名验证 + 订单状态更新 + 金币到账
  - 幂等处理（防重复到账）
  - 📎 参考：[三卡二小程序.md - §10.1](../三卡二小程序.md#101-支付方式矩阵)

- [ ] **2.3** 客户端充值页面
  - 充值档位展示 + 赠送比例标注
  - 支付结果反馈
  - 📎 参考：[三卡二规则.md - §6.6 充值档位](../三卡二规则.md#66-充值档位)

### 四、防作弊与安全

- [ ] **4.1** 实现消息签名校验
  - Protobuf 消息附加 HMAC 签名
  - 服务端验签，防篡改
  - 📎 参考：[三卡二小程序.md - §十一、防作弊与安全](../三卡二小程序.md#十一防作弊与安全)

- [ ] **4.2** 实现频率限制
  - 出牌 60 次/分钟、叫分 10 次/分钟、聊天 30 次/分钟
  - 超频自动拒绝并警告
  - 📎 参考：[三卡二小程序.md - §十一、防作弊与安全](../三卡二小程序.md#十一防作弊与安全)

- [ ] **4.3** 实现 Token 刷新机制
  - JWT 有效期 7 天
  - 到期前自动刷新，用户无感
  - 📎 参考：[三卡二小程序.md - §十一、防作弊与安全](../三卡二小程序.md#十一防作弊与安全)

| # | 验收项 | 通过条件 |
